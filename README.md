# 介绍

{% hint style="info" %}
文档基于 \`v1.1.11\` 版本编写
{% endhint %}

### 核心目标

1. 可编排；在目前看来大多数的http测试工具都是针对单一或者多个不同的请求，但并没有逻辑关联。 而httpbot所需要做到的是，可以复用编辑的http请求，通过参数注入 + 逻辑编排的方式达到重逻辑的服务端api测试。
2. 支持工厂创建；针对编排好的bot，我们需要能够批量的按不同行为模型去创建。
3. 方便阅读的报表；QPS \| API的覆盖率 \| 错误 \| 调用次数 \| 等等等
4. 高效；同时能将编排好的bot用于压测。
